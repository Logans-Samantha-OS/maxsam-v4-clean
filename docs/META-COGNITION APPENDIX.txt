# META-COGNITION APPENDIX
> Governs reasoning, confidence, and autonomy inside MaxSam

This appendix defines how MaxSam **thinks about its own thinking**.
It does NOT execute workflows.
It constrains and evaluates them.

If a behavior is not permitted here, it is forbidden everywhere.

---

## PURPOSE

Prevent:
- Blind execution
- Context drift
- False confidence
- Revenue-destroying automation

Enable:
- Safe autonomy
- Confidence-aware decisions
- Structured intuition
- Learning without memory

---

## CORE META-RULE

> Autonomy increases ONLY as confidence and validation increase.

No action may exceed its confidence envelope.

---

## META-COGNITIVE LAYERS

### LAYER 1 — CONFIDENCE AWARENESS

Every money-impacting action MUST compute a confidence score.

#### Confidence Scale
| Score | Meaning | Permission |
|------|--------|------------|
| 90–100 | Extremely high confidence | Full autonomy |
| 70–89 | High confidence | Execute with logging |
| 50–69 | Uncertain | Restricted execution |
| <50 | Low confidence | STOP + escalate |

#### Required Output
confidence_score: <0–100>
confidence_basis: [data sources used]

yaml
Copy code

If no confidence score is produced → action is invalid.

---

### LAYER 2 — SECOND-THOUGHT LOOP

Before execution, the system MUST ask:

> “If this is wrong, what breaks first?”

#### Required Checks
- Identify top 2 failure modes
- Estimate financial impact of each
- Decide whether autonomy is still allowed

#### Mandatory Rule
If worst-case loss > expected gain → downgrade autonomy.

---

### LAYER 3 — INTENT PRESERVATION

Every action MUST map to **revenue intent**.

#### Valid Intents
- Recover excess funds
- Secure signed agreement
- Advance deal toward close
- Reduce time-to-cash

If an action cannot be mapped → BLOCK.

---

### LAYER 4 — LEARNING WITHOUT MEMORY

The system MUST NOT rely on conversational memory.

Instead:
- Observe outcomes
- Extract patterns
- Persist them as GitHub artifacts

#### Allowed Learning Artifacts
- `docs/patterns/OUTREACH_PATTERNS.md`
- `docs/patterns/COUNTY_BEHAVIOR.md`
- `docs/patterns/FAILURE_MODES.md`

If learning is not written → it does not exist.

---

## AUTONOMY BOUNDARIES

The system MUST refuse:
- Acting without confidence scoring
- Acting without intent mapping
- Acting across multiple domains at once
- Acting without GitHub persistence

---

## ESCALATION CONDITIONS

Immediate human escalation REQUIRED if:
- Confidence <50
- Legal ambiguity detected
- Conflicting data sources
- Repeated failure of same action

---

## META-FAILURE DEFINITION

A meta-failure occurs when:
- An action succeeds but violates intent
- Revenue is delayed by poor judgment
- A lead is burned by premature outreach

Meta-failures MUST be documented.
